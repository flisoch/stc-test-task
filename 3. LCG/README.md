## Метод/алгоритм:
Задача решена по описанному в ссылках методу: 

http://www.reteam.org/papers/e59.pdf

https://security.stackexchange.com/questions/4268/cracking-a-linear-congruential-generator

Для нахождения следующего числа тут нужно больше, чем 4 числа. Минимум 6, но не всегда правильно находится m на малом количестве известных чисел

## Что происходит в программе:
Находятся все неизвестные: m, a, c и seed, а затем генерируется следующее число за последним известным.

В самом генераторе, следующее число которого нужно предсказать, m = 8191, a = c = seed = 7. 

## Запуск/Демо
В терминале запустить `./predictor`

В выводе программы всё описывается
